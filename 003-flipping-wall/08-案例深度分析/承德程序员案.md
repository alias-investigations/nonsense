# 案例深度分析：承德程序员翻墙被罚百万案

## 一、案件事实

### 当事人
- 河北省承德市双桥区居民
- GitHub账号：maliming（ASP.NET Core开源框架ABP贡献排名第二的开发者）

### 时间线
| 时间 | 事件 |
|------|------|
| 2019年9月 | 开始为土耳其软件公司Volosoft提供远程顾问服务 |
| 2019年9月-2022年11月 | 通过GitHub领取任务、编写代码、售后支持、Zoom会议 |
| 2022年9月 | 被当地公安上门调查（起因：怀疑其Twitter账号发布政治评论） |
| 2022年9月 | 手机、笔记本电脑、硬盘被扣押（约一个月后归还） |
| 2023年4月、7月 | 多次被传唤做笔录，详细说明工作情况 |
| 2023年4月20日 | 收到承德市公安局双桥分局"查询财产通知书" |
| 2023年8月 | 收到正式行政处罚决定书 |
| 2023年9月5日 | 申请行政复议，复议部门基本认同公安机关意见 |
| 2023年9月24日 | 在网络上公开此事，引发舆论关注 |
| 2023年10月 | 计划提起行政诉讼 |

### 处罚决定
- **罚款**：200元
- **没收违法所得**：1,058,000元（105.8万元）
- **依据**：暂行规定第6条+第14条，实施办法第7条+第22条

### 工作内容
- 在GitHub上领取公司任务，进行代码编写
- 在售后网站上回答用户问题
- 使用Zoom进行视频会议、远程协助
- 对接公司时偶尔使用翻墙

### 关键细节
- 当事人多次表示：**github.com和公司售后支持网站不需要翻墙就可以访问**
- **编写代码在本地电脑完成也不需要翻墙**
- 但"这些解释没有被采纳"
- 公安机关认为：翻墙违法 → 翻墙之后的收入即是违法所得

## 二、公安机关的法律论证逻辑

### 处罚依据
承德市公安局双桥分局认定当事人违反：
1. 暂行规定第6条："计算机信息网络直接进行国际联网，必须使用国家公用电信网提供的国际出入口信道。任何单位和个人不得自行建立或者使用其他信道进行国际联网。"
2. 暂行规定第14条：违反第6条的处罚（责令停止联网+警告+15000元以下罚款+没收违法所得）
3. 实施办法第7条："我国境内的计算机信息网络直接进行国际联网，必须使用邮电部国家公用电信网提供的国际出入口信道。任何单位和个人不得自行建立或者使用其他信道进行国际联网。"
4. 实施办法第22条：违反第7条的处罚

### 执法机关的推理链条
```
前提1：暂行规定第6条禁止"使用其他信道进行国际联网"
前提2：VPN是"其他信道"
前提3：使用VPN访问境外网站 = "使用其他信道进行国际联网"
结论1：使用VPN翻墙 = 违反暂行规定第6条
前提4：翻墙期间的所有收入 = "违法所得"
结论2：没收全部收入105.8万元
```

### 核心问题
1. **VPN为什么是"其他信道"？** —— 执法机关没有论证，直接将VPN等同于"其他信道"
2. **翻墙期间的收入为什么是"违法所得"？** —— 执法机关认为"翻墙违法 → 翻墙之后的一切收入都是违法所得"
3. **因果关系如何认定？** —— 当事人表示大部分工作不需要翻墙，但公安不采纳

## 三、学界对执法逻辑的逐条反驳

### 反驳1：VPN不是"其他信道"

**王宇扬（2020）的规范解释**：

1996年的暂行规定中，"国际出入口信道"和"其他信道"是在**物理层**面上使用的概念。

**历史解释**：
- 1996年中国互联网刚起步，用户极少，主要为科研机构和高校
- 当时的"非法定信道"指的是**私拉海底光缆、架设卫星天线**等物理接入方式
- VPN技术（1996年发明）在中国普及是2010年以后的事
- 立法者不可能预见到VPN翻墙技术

**体系解释**：
- 实施办法第3条第3款："国际出入口信道，是指国际联网所使用的**物理信道**"
- "其他信道"应与"国际出入口信道"同属物理层面，即"其他**物理**信道"
- VPN是虚拟加密隧道，不是物理信道
- VPN用户的数据仍然通过运营商的物理信道（光缆）传输

**技术事实**：
- VPN只是在应用层（OSI第7层）加密数据
- 物理信道属于物理层（OSI第1层）
- VPN并未建立独立的物理链路
- 断网时VPN无法工作，印证其没有独立建立信道的功能

**结论**：VPN不属于"其他信道"，暂行规定第6条不适用于VPN翻墙。

### 反驳2：翻墙期间的收入不是"违法所得"

**行政处罚法第28条**：
> "违法所得是指实施违法行为所取得的款项。"

**分析**：
- "违法所得"必须与违法行为有**直接因果关系**
- 当事人的收入是**编写代码的劳动报酬**，不是"翻墙"的对价
- 大部分工作（GitHub、编写代码、售后支持）不需要翻墙
- 即使偶尔使用Zoom需要翻墙，Zoom会议只是工作的辅助手段
- 翻墙行为和编写代码之间没有直接因果关系

**类比**：
- 如果一个人开车去上班，路上超速了，他的工资是"违法所得"吗？
- 显然不是。超速是独立的违法行为，工资是劳动的对价。
- 同理，翻墙是独立的（假设的）违法行为，代码编写收入是劳动的对价。

**承德案的荒谬性**：
按此逻辑，任何使用过VPN的人，其此后获得的所有收入都可以被没收。这显然超出了法理和常识的范畴。

### 反驳3：执法机关的"一刀切"逻辑错误

**执法机关的逻辑**：
> "翻墙违法 → 翻墙之后做的一切事情的收入都是违法所得"

**问题**：
1. 这混淆了"手段违法"和"目的违法"
2. 翻墙是手段，编写代码是目的
3. 手段的违法性不能自动传导到目的的合法性
4. 如果目的合法（编写代码不违法），那么手段的违法性不应影响目的的法律评价

**正确的逻辑**：
> "翻墙"（如果违法的话）是独立的违法行为，应独立评价
> "编写代码"是合法行为，其收入是合法劳动所得
> 两者应当分别评价，不能混为一谈

## 六、违法所得认定的法律分析

### 观察者网法律分析（2023-09-27）

核心观点：**105.8万不应被认定为违法所得**。

#### 法律依据
- **行政处罚法第28条**："违法所得是指实施违法行为所取得的款项"
- **最高人民法院研究室意见**：非法经营罪中的"违法所得"应是指获利数额，即全部收入扣除直接用于经营活动的合理支出后剩余的数额
- **工商行政管理机关行政处罚案件违法所得认定办法**：以当事人违法生产、销售商品或者提供服务所获得的全部收入扣除当事人直接用于经营活动的适当的合理支出，为违法所得

#### 分析逻辑
1. **违法行为本身不产生收入**：翻墙本身并不能带来收入。带来收入的是程序员给外国公司提供的服务。
2. **服务本身不违法**：如果这个服务本身不是违法行为，这个服务的报酬，就不应该算成违法所得。
3. **违法所得的正确计算**：即使用最严格的标准，也应计算程序员使用非法VPN获得的**额外获益**作为标准。
   - 合法外网专用信道专线：约2万元/年
   - 共享云服务合法外网访问：几百元/月
   - 三年额外收益（节省成本）：大几千到几万元
   - **翻墙的违法所得应为几千几万元，而非全部收入**

#### 数据支撑
- 2021年涉违法所得行政案件：102件中，30%判决撤销或变更行政处罚决定
- 远高于一般行政诉讼败诉率

#### 程序员的解释
当事人多次表示：
- "github.com和公司的售后支持网站不需要翻墙就可以访问"
- "编写代码在本地电脑完成也不需要翻墙"
- "这些解释都没有被采纳"

#### 胡锡进点评（已被删除）
